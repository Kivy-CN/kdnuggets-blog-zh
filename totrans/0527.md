当涉及到进入数据科学领域时，你需要利用书中的每一个技巧，以便为自己提供一个推动你跨越终点线的优势。

那么，为什么不试着模仿行业中的顶尖人物的习惯呢？

这篇文章不是一种“快速致富”的方法来成为高效的数据科学家。相反，它展示了帮助顶尖数据科学家达到他们现在位置的习惯。

人们常说，数据科学家的价值取决于他们对组织的影响力。这种影响力始于通过培养良好的习惯成为高效而有效的数据科学家。

### 1\. 保持技术的最新状态。

多少当前的数据科学技术仅在过去十年左右的时间里出现？几乎所有的都是。

进入数据科学领域时，你已经注定要终身不断学习。别担心，这并不像听起来那么严峻。

然而，你始终要牢记的是，为了在职场中保持相关性，你需要跟上技术的最新发展。因此，如果你一直用 MATLAB 进行数据分析，可以尝试学习 Python 编程。如果你一直用 Matplotlib 创建可视化，可以尝试使用 Plotly 寻求新鲜感。

**如何实现这一习惯：** 每周花一小时（或尽可能多的时间），试验新技术。通过阅读博客文章找出哪些技术是相关的，选择几个你希望加入你的技术栈。然后，创建一些个人项目，以便最大限度地发挥这些新技术的能力。

### 2\. 保持适当的文档记录。

我总是很幸运地能够阅读和处理那些文档记录糟糕且没有辅助注释的代码，这让我很难理解究竟发生了什么。

我曾经把这归咎于程序员的多愁善感，直到有一天，我意识到这只是一个糟糕程序员的标志。

所有我接触过的优秀程序员都是那些提供清晰、简洁文档来支持他们工作的，同时在他们的程序中加入有用的注释来描述某些代码行的作用的人。这对于使用复杂算法和机器学习模型解决问题的数据科学家尤为重要。

**如何实施这一习惯：** 花一些时间阅读优秀的代码文档或关于如何编写优质代码文档的文章。为了实践，可以为旧的个人项目编写文档，或花时间改进当前项目的文档。由于数据科学世界中的大部分内容基于 Python，查看这篇关于如何记录 Python 代码的优质文章：

[**记录 Python 代码：完整指南 - Real Python**](https://realpython.com/documenting-python-code/)

欢迎阅读你的 Python 代码记录完整指南。无论你是在记录一个小脚本还是一个大型项目……

### 3\. 参与数据科学社区。

开发者是面色苍白的社交孤僻者，他们把自己锁在孤独中编写代码以征服世界的刻板印象已经过时，这种过时的概括并不反映现代技术行业的复杂性。

> “没有人是孤岛。” — 许多数据科学家最喜欢的名言

数据科学的复杂性使得需要一个庞大的支持网络，包括数据科学社区内外的专业人士，以解决各种数据科学家所需的复杂问题。

然而，社区的重要性不仅仅停留在专业层面。随着数据科学领域的扩展，我们有必要帮助铺平未来分析师和工程师的道路，以便他们也能产生影响，并进一步支持其他数据科学家。

随着数据科学领域的“性感”减退，唯一的解决办法是发起一场社区范围的运动，激励行业变得更好。

**如何实施这一习惯：** 成为导师，撰写信息丰富的博客文章，加入数据科学论坛并帮助回答问题，开设一个 YouTube 频道分享你的经验，参加 Kaggle 竞赛和黑客松，或创建课程来帮助未来的数据科学家学习进入行业所需的技能。

### 4\. 定期重构你的代码。

重构是清理代码而不改变其原始功能的过程。虽然重构是软件开发过程中因需求而生的过程，但对数据科学家而言，重构也是一种有用的习惯。

我在重构时的座右铭是 *“少即是多”*。

我发现，当我最初编写代码来解决数据科学问题时，我通常会抛弃良好的编码实践，转而编写在需要时能工作的代码。换句话说，往往会出现大量的“意大利面条代码”。然后，在我的解决方案有效后，我会回去清理代码。

**如何实现这个习惯：** 查看旧代码，问问是否可以用更高效的方式编写相同的代码。如果可以，花一些时间学习最佳编码实践，并寻找可以缩短、优化和澄清代码的方法。看看这篇很棒的文章，概述了代码重构的最佳实践：

[**代码重构最佳实践：何时（以及何时不）进行**](https://www.altexsoft.com/blog/engineering/code-refactoring-best-practices-when-and-when-not-to-do-it/)

代码重构是DevOps软件开发方法中的一个过程，涉及编辑和清理…

### 5\. 优化你的工作空间、工具和工作流程。

有很多提高生产力的IDE扩展，令人惊讶的是，有些人还没有选择优化他们的工作流程。

这个习惯对每个人都非常独特，因此关键在于确定哪些工具、工作空间和工作流程使你成为最有效率的数据科学家。

**如何实现这个习惯：** 每年一次（或者更频繁，如果这样对你更有效），评估你整体的工作效率和效果，并确定可以改进的地方。也许这意味着早晨一开始就处理你的机器学习算法，或者坐在运动球上而不是椅子上，或者为你的IDE添加一个新的扩展，以便对你的代码进行静态检查。尝试不同的工作空间、工具和工作流程，直到你进入最佳状态。

### 6\. 关注理解业务问题。

从我所见，数据科学是75%理解业务问题和25%编写模型以找出解决方案。

编码、算法和数学是简单的部分。理解如何实施它们以解决特定的业务问题则不那么简单。通过花更多时间理解业务问题和你要解决的目标，剩下的过程将会顺利得多。

要了解你所在行业面临的问题，你需要进行一些调查，以便获得支持你解决问题的背景知识。例如，你需要了解某个业务的客户需求是什么，或工程公司试图达成的具体目标。

**如何实现这个习惯：** 花时间研究你工作的具体公司及其所在行业。写一份备忘单，包含公司的主要目标以及它在特定行业中可能面临的问题。不要忘记包括你可能想用来解决业务问题的算法或未来可能有用的机器学习模型的想法。每当你发现有用的信息时，添加到这份备忘单中，很快你就会拥有一份行业相关的宝贵资料。

### 7\. 采用极简主义风格。

不，不是在生活中。在你的代码和工作流程中。

人们常常争论，[最优秀的数据科学家](https://www.iotforall.com/8-data-science-career-skills) 使用最少的代码、最少的数据和最简单的算法来完成工作。

尽管我提到的极简主义并不意味着你立即就要假设稀缺性。通常，当有人讨论代码中的极简主义重要性时，会导致人们尝试开发仅用几行代码的夸张解决方案。停止这种做法。是的，这确实令人印象深刻，但这真的能最有效地利用你的时间吗？

相反，一旦你对数据科学概念感到熟悉，开始寻找优化代码的方法，使其简单、干净、简短。使用简单的算法完成任务，并且不要忘记编写可重用的函数来消除冗余。

**如何实施这个习惯：** 随着你作为数据科学家的进步，开始推动自己编写更高效的解决方案，写更少的代码，使用更简单的算法和模型来完成任务。学习如何缩短代码而不减少其有效性，并留下大量注释来解释缩写版本的代码是如何工作的。

### 8\. 使用函数来消除复杂性和冗余。

我首先要承认，当我第一次编写数据分析代码时，我严重忽略了函数。我的IDE中充满了意大利面条式的代码，因为我在努力理清不同的分析思路。如果你查看我的代码，你可能会认为它已经烂到无法拯救，主动提出把它拖到谷仓后面，结束它的痛苦。

一旦我成功拼凑出一个还算不错的结果，我会回过头来尝试修复相当于一场严重的事故。通过将我的代码打包成函数，我迅速消除了不必要的复杂性和冗余。如果这就是我对代码所做的唯一操作，那么我已经将它简化到一个我可以重新审视解决方案并理解我如何达到那个点的程度。

**如何实施这个习惯：** 不要忘记编写代码时函数的重要性。人们常说，最好的开发者是懒惰的开发者，因为他们找到了解决方案而无需过多工作的方法。在你编写了一个解决方案之后，回过头来将冗余或复杂的代码打包成函数，以帮助组织和简化代码。

### 9\. 应用测试驱动开发方法。

[测试驱动开发](https://towardsdatascience.com/tdd-datascience-689c98492fcc)（TDD）是一种软件开发原则，专注于通过不断测试的增量改进来编写代码。TDD遵循“红色、绿色、重构”系统，鼓励开发人员构建测试套件、编写实现代码，然后优化代码库。

数据科学家可以成功实施TDD来生成分析管道、开发概念验证、处理数据子集，并确保在开发过程中功能代码不会被破坏。

**如何实现这个习惯：** 学习测试驱动开发（TDD），并确定这种技术是否能为你的工作流程带来帮助。TDD不是解决所有问题的完美答案，但如果实施得当，它可以非常有用。查看这篇文章，它很好地描述了TDD，并提供了如何将其应用于数据科学项目的示例：

[**如何在数据科学工作流程中使用测试驱动开发**](https://towardsdatascience.com/tdd-datascience-689c98492fcc)

数据科学家和机器学习工程师应从软件开发人员那里学到的另一件事

### 10. 进行小而频繁的提交。

是否曾经提交一个拉取请求后，电脑上出现了大量错误消息和问题？我经历过。这真的很糟糕。

当你觉得想用拳头去迎接做了如此大提交的人时，深呼吸一下，记住这个人显然没有花时间去培养良好的习惯。

团队开发软件的黄金法则是什么？进行小而频繁的提交。

**如何实现这个习惯：** 养成频繁提交代码更改并同样定期进行拉取请求以获取最新代码的习惯。你或其他人所做的每一个更改都有可能破坏整个项目，因此重要的是要进行小的更改，这些更改易于恢复，并且通常只影响项目的一部分或层次。

### 11. 把自我发展放在优先位置。

根据你问的人不同，业内要么数据科学家太多，要么太少。

无论行业是趋于饱和还是贫瘠，你都会与大量高度合格甚至过度合格的候选人竞争一个职位。这意味着在申请工作之前，你需要已经培养了自我提升的习惯。如今，每个人都对技能提升充满热情，这也是有充分理由的。这一趋势对数据科学家也不例外。

**如何实现这个习惯：** 制定技能清单，看看你在招聘广告中列出的要求下表现如何。你是一个能够高效使用Keras、NumPy、Pandas、PyTorch、TensorFlow、Matplotlib、Seaborn和Plotly等相关库的Python专家吗？你能写一份详细说明你最新发现的备忘录，并解释这些发现如何将公司效率提高25%吗？你是否能舒适地与团队合作完成项目？识别出自己的不足，并找到一些好的在线课程或资源来提升你的技能。

### 12. 以终为始开始一个项目。

在[*高效能人士的七个习惯*](https://www.franklincovey.com/the-7-habits/)中，斯蒂芬·柯维讨论了“以终为始”的原则。

为了有效地将这一点应用于数据科学项目，你需要在项目的规划阶段问自己项目的预期结果是什么。这将帮助塑造项目的路径，并为你提供一个需要达到的结果的路线图。除此之外，确定项目的结果将使你对项目整体的可行性和可持续性有一个概念。

**如何实施这一习惯：** 在每个项目开始时进行规划会议，明确你希望在开发周期结束时达到的目标。确定你将尝试解决的问题，或你试图收集的证据。然后，你可以开始回答可行性和可持续性的问题，这将塑造你的项目的里程碑和结果。之后，你可以开始编写代码和机器学习模型，并制定明确的计划来指导你完成项目。

### 13\. 理解以便你能被理解。

在未能成功准备关于为什么自旋-V2 粒子遵循费米-狄拉克统计的大学生讲座后，[理查德·费曼](https://magazine.caltech.edu/post/feynman-at-100)曾著名地说：“我无法将其简化到大学生水平。这意味着我们真的不理解它。”被称为“伟大的解释者”的费曼留下了数据科学家只能希望效仿的遗产。

数据科学，即利用数据讲述引人入胜的故事的艺术，只有当讲述者理解他们想讲的故事时才会成功。换句话说，你的任务是理解，以便你能被理解。及早养成理解你想完成的任务的习惯，这样你才能以适当的程度与他人分享，将使你成为房间里最有效的数据科学家。

**如何实施这一习惯：** 使用[费曼技巧](https://blog.doist.com/feynman-technique/)来深入理解你试图发现的概念和你试图解决的问题。这种方法与数据科学过程中的数据分析和结果解释非常契合。简而言之，你将你的解释精炼到可以用简单的、不使用行话的术语来解释，以便任何人都能理解。

### 14\. 阅读研究论文。

在一个由硕士和博士主导的领域，研究论文常用于分享行业新闻和见解。

[研究论文](/2021/02/reading-papers-effective-data-scientist.html) 是了解他人如何解决问题、拓宽视野和跟上最新趋势的有用方式。

**如何培养这一习惯：** 每周挑选一两篇与你当前工作相关的或你感兴趣的技术领域的研究论文进行阅读。尽量每周留出时间进行文献回顾，把它作为优先事项。熟悉阅读研究论文的[三遍法](https://web.stanford.edu/class/ee384m/Handouts/HowtoReadPaper.pdf)，这有助于你快速收集相关信息。为了真正巩固你对论文的理解，尝试将从阅读中学到的东西应用到个人项目中，或与工作同事分享你所学到的内容。

### 15. 保持开放心态。

数据科学的世界正在迅速变化，从使用的技术到达成的目标都在不断更新。不要成为那个固步自封、不愿改变的数据科学家。

不仅保持开放的心态迫使你不断提升自己的专业能力，而且还能让你在迅速变化的行业中保持相关性，否则你会在落后时被淘汰。

**如何培养这一习惯：** 每当有新技术或实践成为新闻时，进行试用，看看这些新技术或实践能带来什么。即使只是阅读文档，你也能保持对行业变化趋势的了解。此外，你可以将对技术的看法带到公司，帮助他们应对技术变革和进步。成为办公室中对变化保持敏感的人可以帮助你走在前沿，也可以帮助你引导团队和公司找到更好、更高效的解决方案。

在数据科学职业生涯的任何阶段，培养良好的习惯使你能够发挥潜力，成为团队中有效的成员，对解决问题产生重大影响。

现在正是设定未来成功目标的最佳时机。
